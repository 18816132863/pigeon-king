V24.7 End-side Global Serial Executor

This is an incremental patch for the current V24.6 project. Do not run it as a standalone clean release.

Main change:
- All end-side capabilities must serialize during multi-action execution.
- The rule is global, not limited to alarm + hiboard + chat cron reminders.
- call_device_tool / GUI device action / app action / notification / alarm / calendar / file / settings must use the device_action_queue / EndSideGlobalSerialExecutor.
- Timeout becomes timeout_pending_verify, not device_offline.
- Duplicate side effects are skipped by idempotency key.

Commands:
/usr/bin/python3 scripts/v24_7_end_side_global_serial_smoke.py
/usr/bin/python3 scripts/v24_7_end_side_global_serial_gate.py
