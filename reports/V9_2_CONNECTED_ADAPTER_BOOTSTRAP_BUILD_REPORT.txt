## 解决的问题

针对真实小艺连接端出现的情况：

- runtime_available = true
- call_device_tool_available = true
- adapter_loaded = false
- L0 route 全部 timeout

V9.2 新增 adapter bootstrap 前置门：

session_connected=true
→ runtime_bridge_ready 检查
→ adapter_bootstrap
→ adapter_loaded=true/partial/failed
→ probe_minimal_service
→ safe_route_test

如果 adapter_loaded=false，不再直接跑 L0 route，而是进入恢复链：

reload_adapter
→ rebind_call_device_tool
→ probe_minimal_service
→ switch_to_direct_tool_bridge
→ service_specific_probe
→ human_action_required
